# ADR-0002: C3K Desktop Client совмещает Storage Node и локальный клиент c3k.ton

## Статус

Принято

## Дата

2026-03-18

## Контекст

Пользователю не должно требоваться:

- устанавливать сторонний TON browser
- вручную подключать публичные TON proxy
- разбираться в инфраструктурных деталях ради открытия `c3k.ton`

Одновременно у проекта уже появляется desktop-клиент для storage node.

## Решение

`C3K Desktop Client` выполняет две роли:

- `C3K Storage Node`
- локальный клиент открытия `c3k.ton`

## Причины

- уменьшается friction для пользователя
- нет необходимости плодить два разных desktop-приложения
- локальный gateway логично живет там же, где и storage runtime

## Последствия

### Плюсы

- один desktop entrypoint для power users
- проще объяснить продукт
- сильнее связка между storage program и TON-native поверхностью

### Минусы

- десктоп-клиент становится сложнее
- выше ответственность на стабильность desktop runtime
- TON Site access и storage node failures могут влиять на один и тот же клиент

## Следствие для кода

- нужно проектировать `Desktop Core` модульно
- `storage-daemon` wrapper и `TON Site gateway` wrapper должны быть независимыми сервисами внутри одного приложения
