# Roadmap Layer

В этой папке теперь есть три уровня управления проектом.

## 1. Strategic Roadmap

- [`production-roadmap.md`](/Users/culture3k/Documents/GitHub/c3k-blog/Documentation/roadmap/production-roadmap.md)

Нужен для ответа на вопрос:

- к каким фазам мы идём
- какие большие workstreams должны быть завершены до реального production

## 2. Sprint Board

- [`sprint-board.md`](/Users/culture3k/Documents/GitHub/c3k-blog/Documentation/roadmap/sprint-board.md)

Нужен для ответа на вопрос:

- какой спринт идёт сейчас
- что мы обязаны доставить в этом спринте
- что идёт следующим спринтом

## 3. Product Capability Status

- [`project-status-checklist.md`](/Users/culture3k/Documents/GitHub/c3k-blog/Documentation/roadmap/project-status-checklist.md)

Нужен для ответа на вопрос:

- что уже умеет продукт по пользовательским и бизнес-сценариям
- какие user stories ещё не доставлены

## Как этим пользоваться вместе

1. `Sprint board` использовать каждый день
2. `Project status checklist` обновлять, когда меняется продуктовая ценность
3. `Production roadmap` обновлять, когда меняется стратегия выхода в production
