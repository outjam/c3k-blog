# User Cases by Section

## 1. Новости

### User case

Как пользователь, я хочу открыть приложение и сразу увидеть:

- свежие релизы
- активность людей, на которых я подписан
- посты и релизы в одном понятном потоке

### Ключевые сценарии

- открыть карточку релиза
- открыть карточку поста
- перейти в профиль автора
- понять, куплен ли релиз уже мной
- увидеть, есть ли NFT

## 2. Релизы

### User case

Как пользователь, я хочу изучать каталог музыки и быстро понимать:

- кто артист
- сколько стоит релиз
- в каких форматах он доступен
- есть ли он у меня в коллекции
- куплен ли он частично

### Ключевые сценарии

- открыть релиз
- добавить в избранное
- перейти к артисту
- фильтровать каталог

## 3. Релиз

### User case

Как пользователь, я хочу покупать музыку максимально гибко:

- купить один трек
- купить весь релиз
- выбрать формат
- позже улучшить релиз до NFT

### Ключевые сценарии

- выбрать формат
- купить отдельный трек
- купить релиз целиком
- увидеть already owned state
- открыть комментарии и реакции
- заминтить NFT, если релиз полностью куплен

### Edge cases

- куплен только один трек
- куплен релиз в одном формате, но не в другом
- релиз не поддерживает mint
- у пользователя не привязан TON wallet

## 4. Профиль пользователя

### User case

Как пользователь, я хочу видеть:

- свою идентичность
- свои подписки и подписчиков
- свою коллекцию
- свои NFT улучшения
- свои награды
- свой баланс

### Ключевые сценарии

- открыть настройки
- поделиться профилем
- посмотреть коллекцию
- посмотреть награды
- открыть списки подписчиков и подписок

## 5. Публичный профиль пользователя

### User case

Как другой пользователь, я хочу:

- открыть чей-то профиль
- подписаться на него
- увидеть его коллекцию и награды
- понять, это артист или обычный пользователь

## 6. Настройки

### User case

Как пользователь, я хочу управлять:

- своим user profile
- языком
- темой
- видимостью покупок
- режимом доступа к artist application

## 7. Подача заявки на артиста

### User case

Как пользователь, я хочу стать артистом в приложении.

### Сценарий

- заполнить анкету
- приложить ссылки и TON wallet
- получить обратную связь от админа
- после approve получить доступ к студии

## 8. Студия артиста

### User case

Как артист, я хочу видеть в одном месте:

- сколько у меня релизов
- сколько я заработал
- сколько доступно к выводу
- как редактировать публичный профиль артиста
- как создать новый релиз

## 9. Выплаты артисту

### User case

Как артист, я хочу:

- понимать, сколько средств доступно
- знать, почему часть средств еще в hold
- отправить запрос на вывод
- получить статус по запросу

### Важные ограничения

- без TON wallet payout request невозможен
- до минимума payout невозможен
- до окончания hold payout невозможен

## 10. Баланс и пополнение

### User case

Как пользователь, я хочу:

- быстро пополнить баланс
- использовать Stars или TON
- видеть свой текущий баланс
- понимать, для чего он нужен

## 11. Админка

### User case

Как владелец сервиса, я хочу:

- видеть заявки артистов
- модерировать выплаты
- управлять каталогом
- управлять промо
- управлять настройками

## 12. Блокчейн сценарии

### User case

Как пользователь-коллекционер, я хочу:

- подключить TON wallet
- купить релиз
- улучшить релиз до NFT
- видеть этот NFT в профиле и коллекции

### User case артиста

Как артист, я хочу:

- чтобы мои релизы могли быть mintable
- понимать, какие релизы допускают NFT upgrade
- в будущем использовать NFT как collector utility слой
