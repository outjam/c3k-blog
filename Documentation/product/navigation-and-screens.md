# Navigation and Screens

## 1. Главная карта маршрутов

### Главные экраны с Tab Bar

- `/` — Новости
- `/shop` — Релизы
- `/profile` — Мой профиль

Именно на этих трех экранах показывается нижний tab bar.

### Вторичные экраны

- `/balance` — пополнение баланса
- `/downloads` — библиотека файлов и delivery requests
- `/storage` — программа `C3K Storage`
- `/shop/[slug]` — страница релиза
- `/shop/artist/[slug]` — публичная страница артиста
- `/profile/[slug]` — внешний публичный профиль пользователя
- `/profile/edit` — настройки
- `/studio` — отдельная студия артиста
- `/post/[slug]` — страница поста
- `/shop/cart` — корзина
- `/orders/[id]` — детальная страница заказа
- `/admin` — админ-панель
- `/admin/artists` — админка артистов
- `/admin/showcase` — управление витринами
- `/tools` и `/tools/track-cover` — внутренние инструменты

## 2. Верхний App Frame

Общий shell состоит из:

- header `Culture3k Network / Elite Music Community`
- content area
- global player
- mobile tab bar на главных экранах

На вторичных экранах нижний tab bar скрыт, а возврат обеспечивается через back action.

## 3. Экран `Новости`

### Назначение

Это социальная точка входа в приложение.

### Что там должно происходить

- выдача релизов и постов
- два таба контента
- свайп между вкладками
- sticky tabs при скролле
- карточки релизов с персонализированным ownership статусом

### Что пользователь ожидает увидеть

- кто выпустил релиз
- есть ли релиз у него в коллекции
- куплен ли он частично
- есть ли NFT
- сколько реакций и комментариев

## 4. Экран `Релизы`

### Назначение

Каталог релизов и артистов.

### Состав

- поисковая фильтрация внутри каталога
- quick filters
- rail артистов
- карточки релизов
- персональные ownership markers

### Требование к UX

Каталог должен быть не generic e-commerce grid, а музыкальной витриной.

## 5. Экран `Профиль`

### Назначение

Центральный экран идентичности пользователя.

### Блоки

- hero
- социальные метрики
- wallet card
- summary по файлам и delivery activity
- кнопки `Настройки / Поделиться`
- `Коллекция`
- `Награды`
- `Студия` только для подтвержденного артиста

### Особенности

- tabs sticky
- swipe между tab pages
- collection включает как полные релизы, так и частично купленные

## 6. Экран внешнего профиля

Маршрут: `/profile/[slug]`

Это социальный профиль пользователя, а не artist admin screen.

Он нужен для:

- просмотра коллекции
- подписки
- просмотра наград
- социальной идентичности

## 7. Экран артиста

Маршрут: `/shop/artist/[slug]`

Это публичная storefront/profile страница артиста.

Она нужна для:

- bio артиста
- каталога его релизов
- поддержки артиста
- подписки на артиста

Это не то же самое, что обычный user profile.

## 8. Экран релиза

Маршрут: `/shop/[slug]`

Это ключевой transactional screen.

### На нем должно быть

- cover и identity артиста
- формат-релиз selector
- tracklist как главный фокус
- покупка отдельных треков
- покупка релиза целиком
- owned state
- NFT upgrade / mint
- реакции и комментарии

### Главный UX-приоритет

Релиз должен ощущаться как музыкальная страница, а не как карточка товара интернет-магазина.

## 9. Экран `Настройки`

Маршрут: `/profile/edit`

Это не просто редактирование, а settings hub.

Там должны жить:

- редактирование user profile
- приватность
- theme
- language
- artist application
- история операций
- account preferences

## 10. Экран `Студия`

Маршрут: `/studio`

Это отдельная админка артиста.

Внутри нее:

- dashboard
- редактирование artist profile
- управление релизами
- выплаты

Текущий продуктовый принцип:

- вкладка `Студия` внутри профиля = короткий summary
- `/studio` = полноценный кабинет артиста

## 11. Экран `Баланс`

Маршрут: `/balance`

Назначение:

- пополнение через Stars
- пополнение через TON
- привязка TON wallet
- работа с promo codes

## 12. Экран `Файлы`

Маршрут: `/downloads`

Назначение:

- отдельный consumer-facing центр загрузок
- история выдач релизов и треков
- фильтрация по статусам
- retry для failed/pending requests
- переход обратно к релизу

Это не storage admin и не программа `C3K Storage`, а пользовательская post-purchase библиотека.

## 13. Экран `C3K Storage`

Маршрут: `/storage`

Назначение:

- вход в программу `C3K Storage`
- статус участия
- node/desktop/TON Site контур
- служебная история delivery requests как часть storage layer

## 14. Cart и Orders

### `/shop/cart`

Остаток более классического checkout flow.

### `/orders/[id]`

Страница конкретного заказа.

Эти экраны полезны, но продуктово нужно решить:

- будут ли они основным путем покупки
- или покупка целиком уйдет в inline purchase прямо из релиза

## 15. Служебные зоны

### `/admin`

Backoffice владельца и команды.

### `/tools`

Внутренние мини-инструменты.

Они не должны перегружать consumer-путь.
